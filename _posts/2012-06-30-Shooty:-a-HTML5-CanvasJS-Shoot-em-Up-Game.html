Motivation

Shooty started out as a training for the Canvas-HTML-element and how to manipulate it via JavaScript. But soon, it got out of control :)

I did not start from scratch, I had some artwork laying around from various older game projects. Most of the code has been written in a marathon programming week during vacation.

Goals

My primary goal was to learn about the canvas element in modern browser and if it could provide everything for writing simple games. I wanted to find out weather it could handle multiple bitmap operations, keyboard handling and event handling.

This goal has been reached, by writing Shooty.

My next goal is to re-make Shooty, to make it easy to add new artwork, functions and levels and to clean the code.

Artwork

The artwork has been created completely with GIMP[1] and Blender[2]. Most of the artwork is still work in progress and should be considered as a placeholder for future artwork.


Project-Status

The game is incomplete. It lacks some basic features e.g. sound effects. The source code has grown and is a mess.

Next steps

Freezing the project where it is right now and refactoring the code are the next steps ahead.

Like with Dubnotes, I could write unit-tests on the existing code and redesign the app from there, but I think in this case it is inappropriate.

Reasons why I think it has be re-done are:
  • Unit tests can not cover all the graphical input / output quite well, I have to split the UI-code from the game engine, creating a new layer of abstraction.
  • I have to write the tests from ground up and the code needs to split up into testable functions.
  • The coding style is bad.

From the current state I can easily create an application design and then write the new code within the TDD framework fast.
